Réponse
Surapprentissage

Le modèle apprend trop les données d'entraînement et ne généralise pas sur de nouvelles données.

Comment ça fonctionne ?
  1. L'application sélectionne une carte via sélection pondérée (random.choices)
  2. L'utilisateur lit la question et révèle la réponse
  3. Il indique s'il connaissait ou non la réponse
  4. La probabilité d'apparition de la carte est mise à jour immédiatement
  5. Les statistiques du jour sont enregistrées en base
Algorithme adaptatif de probabilités

La probabilité d'apparition de chaque carte est mise à jour dynamiquement après chaque réponse. Les cartes mal maîtrisées reviennent plus souvent.

✅
× 0.7
Bonne réponse
Probabilité réduite
prob = prob × 0.7

La carte est maîtrisée → elle apparaît moins souvent pour laisser place aux cartes difficiles.

❌
× 1.3
Mauvaise réponse
Probabilité augmentée
prob = prob × 1.3

La carte est difficile → elle revient plus fréquemment jusqu'à être maîtrisée.

Contraintes système
0.1 ≤ P ≤ 1.0

La probabilité est bornée pour éviter qu'une carte disparaisse totalement ou monopolise les sessions.

Mise à jour immédiate via
PRAGMA wal_checkpoint(FULL)

Architecture SQLite — 3 tables
TABLE Themes
├ ID INTEGER PRIMARY KEY
└ Theme TEXT

Organise les flashcards par domaine. Suppression protégée par clé étrangère (ON DELETE RESTRICT).

TABLE Cards
├ ID INTEGER PRIMARY KEY
├ Question TEXT
├ Reponse TEXT
├ Probabilite REAL CHECK(0.1→1)
└ id_theme INTEGER FK

Cœur du système. La probabilité est bornée en base via une contrainte CHECK.

TABLE Stats
├ ID INTEGER PRIMARY KEY
├ Bonnes_Reponses INTEGER
├ Mauvaises_Reponses INTEGER
└ Date DATE

Une ligne par jour. Les stats sont agrégées par date pour suivre la progression.

Fonctionnalités de l'application
🔁
Révision adaptive

Sélection pondérée via random.choices. Les cartes difficiles (probabilité élevée) sont plus souvent tirées.

➕
Ajout de cartes & thèmes

Interface pour créer de nouvelles flashcards avec vérification des doublons avant insertion en base.

📊
Statistiques quotidiennes

Chaque réponse est enregistrée par date. Historique consultable et réinitialisation des probabilités à 0.5 possible.

📂
Navigation par thème

Affichage des cartes filtrées par thème avec leur probabilité actuelle pour suivre la maîtrise par domaine.

🔍
Débogage des probabilités

Vue complète des probabilités triées par ordre décroissant pour inspecter l'évolution de chaque carte.

🗑️
Gestion des thèmes

Ajout et suppression de thèmes. Suppression impossible si des cartes y sont associées (intégrité référentielle SQLite).

Pipeline d'une session de révision
Choix du thème

Sélection dans la liste des thèmes disponibles

Tirage pondéré

random.choices avec poids = probabilités normalisées

Révélation

Question affichée → clic pour voir la réponse

Mise à jour

Probabilité × 0.7 ou × 1.3 + WAL checkpoint

Stats

Enregistrement date + bonnes/mauvaises réponses
